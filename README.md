# marine-detect

해상 경계 이상탐지 — **탐지(YOLO) 기반 파이프라인.** MA-PDM(재구성 VAD)이 이 과제
(작은 객체 + 동적 배경 + 외형 이상)에 부적합하다는 W3 진단 이후의 후속 파이프라인.

## 왜 탐지 기반인가 (W3 결론 요약)
- 재구성 VAD: 혼합 spot 13% / 단일 spot 60%(천장) — 재구성 오차가 파도·팬(동적 배경)에 지배됨.
- 같은 데이터의 크롭 지도학습 분류: **95%** — 정보는 픽셀에 살아있고, 재구성이 못 꺼낼 뿐.
- → 탐지기가 객체를 짚고(배경 배제), 크롭 단위로 판정. 팬 카메라 가정도 사라짐.

## 파이프라인 (3-stage)
1. **탐지** — YOLO11 계열. 71858 bbox로 학습(어선/상선/군함). 프레임 단위라 정지카메라 가정 불필요.
2. **이상 판정 헤드** — 이상 정의에 따라 둘 중:
   - (a) 알려진 희귀클래스: 탐지기가 군함 직접 분류 → 끝.
   - (b) 열린집합: 크롭 임베딩을 정상(어선·상선) 프로토타입과 거리 비교 → outlier.
3. **데모 안정화** — 경량 트래커(ByteTrack) + 시간 스무딩. (선택) 에고모션 보정으로 궤적 이상 확장.

## 데이터
- 라벨 원본: `../data/labels_train/*.json` (41,972프레임, 전 클래스). 클래스 0=어선 1=상선 2=군함 4=복합.
- 클래스별 bbox 파생본: `../aug/out_train/boxes_02.csv` (어선/군함만).
- 분할: **clip 단위**(누수 방지). clips.csv 863클립 = 정상 616 / 군함 247.
